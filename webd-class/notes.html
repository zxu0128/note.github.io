    CSS: Cascading Style Sheets

    • Selector: a selector designates exactly which element with in HTML to target and apply styles
      1. Type Selector: Target elements by their element type
      2. Class Selector: Select an element based on the element's class attribute value
      3. ID selector: more precise than class selectors, id attribute values can only be used once per page
    • Property: a property determines the styles that will be applied to the element (such as: font-size, color)
    • Values: determine the behaviour of a property

    2.What Is Cascading?

    What if one selector is defined in two external files?

    • The rules from the most recent file have precedence (aka, the last one listed has the most precedence)

    What if one Selector has more than one rule in the same file?
    • It will always use the last one added

    Override Rules!: The different values cascade between the options style attribute, internal rules, external rules, and browser default. with the Style attribute in a tag override the Internal Styling i.e. rules specified in the <head> section

    To override later rules: use "!important"

    8.0 Shay Howe: READING NOTES

      Common Selectors (type, class, ID Selectors)
    • Type selector: identifies an element based on its type
    • Class selector: identifies an element based on class attribute value
    • ID selector: identifies an element based its ID attribute (unique and should only be used once per page)

    8.1 Child Selectors

    • Descendant Selector: Select an element that resides anywhere within an identified ancestor element (e.g. article h2, select h2 that falls in the article element - doesn't have to be direct)

    • Direct Child Selector: Place an ">" sign between the parent and child selector. (e.g.: article > p, is a direct child selector only identifying p elements that fall directly within an article element).

    8.2 Sibling Selectors

      General Sibling Selector
    • Select elements that share the same parent, the two elements are connected by the tilde (~) character. (e.g. h2 ~ p selects all p elements that follow, and share the same parent, of any h2 elements).
      • Adjacent Sibling Selector
    • Select a sibling element that directly follows after another sibling element. Use a + character between the two sibling elements. (e.g. h2 + p selects p elements directly follow after h2 elements that share a same parent element.

    8.3 Attribute Selector

    Attribute Present Selector:To select an element based on if an attribute is present or not, include the attribute name in square

    Attribute Equals Selector:To identify an element with a specific, and exact matching, attribute value

    Attribute Contains Selector:Can find an element based on part of an attribute value, but not an exact match (use asterisk character, *)

    Attribute Begins With Selector:Select element based on if an attribute value begins with something (use a circumflex accent ^ right after the attribute name)

    Attribute Ends With Selector:Select element based on if an attribute value that ends with something (use a dollar sign $ right after the attribute name)

    Attribute Spaced Selector:At times attribute values may be spaced apart, in which only one of the words needs to be matched in order to make a selection. In this event using the tilde character ~ right after the attribute name.

    Attribute Hyphenated Selector:Selects an element if the given attribute value is hyphen-separated and begins with the word stated (use | right after the element name.

    9. PSEUDO-CLASSES

    Pseudo-classes: Begin with the colon character ":"

    • Link Pseudo-classes (define a link has been visited or not): such as a:link {...} and a:visited: {...}
    • User Action Pseudo-classes: such as a:hover{}, a:active{}, a:focus{}
    • User Interface State Pseudo-Classes: include: enabled, :disabled, :checked, and :indeterminate.

    • Structural & Position Pseudo-classes:
        • BASED ON PARENT-CHILD
      1. :first-child - Select an element that is the first-child within its parent
      2. :last-child - Select an element that is the last-child within its parent
      3. :only-child - Select an element that is the only-child within its parent
        4. (e.g: li: first-child {...})

        BASED ON TYPE
      1. :first-of-type (first-of-type pseudo-class will select the first element of its type within a parent)
      2. :last-of-type (select the last elementof its type within a parent)
      3. :only-of-type (select an element if it is the only of its type within a parent)
        4. e.g.: p:first-of-type {...}, img:only-of-type {...}

        BASED ON A NUMBER OR AN ALGEBRAIC EXPRESSION
      1. :nth-child(n)
      2. :nth-last-child(n)
      3. :nth-of-type(n)
      4. :nth-last-of-type(n)

      e.g.: the li:nth-child(3n) selector will identify every third list item within a list. Using the expression this equates to 3×0, 3×1, 3×2, and so forth. li:nth-child(4n+7) selector will identify every fourth list item starting with the seventh list item.

      The :nth-child(n) works from the beginning of the document tree while the :nth-last-child(n) works from the end of the document tree.

    ***Reading: Using Multiple <h1>s

    Use one set of <h1> tags per sectioning root or content section.
    There should always be a <h1> level heading between the opening <body> tag and the first content section, to label the overall document.
    When a <h1> level heading is to be used to label a content section, it should be the first heading that appears in the section, because the first heading is always interpreted as the section's label.
    If a <h1> level heading is used to label a content section, any other headings used in that section should be H2 or lower in order to create an accurate document outline.

    12.Setting Background & Gradients

    • Background image - similar tp adding images and urls:
      div {
      background-image: url("alert.png");
      }
    • Background repeat: By default, a background image will repeat indefinitely, both vertically and horizontally, unless otherwise specified. Therefore "background-repeat" is used to change this (values include: repeat, repeat-x, repeat-y, and no repeat)
    • Background-position: requires two values, a horizontal offset (the first value) and a vertical offset (the second value). If only one value is specified, that value is used for the horizontal offset and the vertical offset will default to 50%. (moving the background image from the left top corner of the element, length values specifically will be in relation to that corner)

    NOTE: !SHORTHAND! The background-color, background-image, background-position, and background-repeat properties may be rolled up into a shorthand value for the background property alone. The order of these properties commonly falls as background-color, background-image, background-position, and then background-repeat.
